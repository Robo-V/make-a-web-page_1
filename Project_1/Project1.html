The Basics of the Web

The Web is collection of hyperlinked documents written in Hypertext Markup Language (HTML). The Web currently contains more than thirty billion pages.

The Web is made up of:

  • You and your computer
  • Browser; a program that runs on your computer to display files found on the web
  • The Internet: The world's largest computer network
  • HTTP; the main protocol of the web
  • Servers; computers that host the files that make up the web

Hypertext Markup Language (HTML)

HTML is made up of:

  • Text content; what you see
  • Markup; what it looks like
  • References to other documents; e.g. images and videos
  • Links to other pages

HTML Structure

Using proper HTML structure is essential, as incorrect structure may result in the inability of a search engine to read the page. The structure is based on a hierarchy of elements and is refered to as the Document Object Model (DOM), or the document tree.

HTML Vocabulary

  • Tags; basic HTML syntax
  • Elements; everything within a set of opening and closing tags
  • In-line elements; all text rendered on a single line
  • Block elements; essentially draws an "invisible box" around the text, with line breaks and spacing
  • Attributes; a property of an HTML element. For example, including href="www.google.com" in an anchor tag sets the href attribute to the Google URL.

More on HTML tags:

  • Opening tags contained within angled brackets (the "less than" and "greater than" symbols)
  • Closing tags are preceded by the "/" and contained within the angled brackets
  • Void tags do not require a closing tag (the break tag, for example)
  • The "div" tag is highly useful for organizing HTML, as it helps divide elements into sections where CSS styling can be applied to the sections. This will allow for cleaner code, make it easier to change or update and avoid repetition.

Some basic examples:

  • Bold: b, an in-line element
  • Emphasis: em, this will italicize text, an in-line element
  • Anchor: a, used for making links, an in-line element
  • Image: img, embedding or attaching images, an in-line element
  • Break: br, used to render multiple lines, an in-line element, despite rendering seperate lines
  • Paragraph: p, formats text as paragraph, a block element

A helpful visual: imagine a website as a house

  • HTML is the structure: where are the walls placed, what room is the living room
  • CSS is the style: what color are the walls or carpeting, what kind of furniture you have
  • Javascript is the interactive components: garage door opener, tv remote; they change something within the house
The browser interprets and displays the HTML structure by turning HTML tags into elements in the DOM (tree); each tag creates an element in the DOM that the browser uses to create the page. An element is created from a starting tag to the ending tag, with everything in between as content.

Cascading Style Sheets (CSS)

CSS is code written to control the style of the HTML elements. There are three ways that CSS can be applied:

  • Written into the head of the HTML; may be used for small projects
  • Written in-line with your HTML; very bad practice, as this leads to repeating code and makes it very difficult to change/update
  • Link HTML code to separate CSS file; the preferred method, especially when working on bigger projects. Allows for easier updating than other methods and avoids repeating code.

Inheritance

Inheritance is a key feature of CSS, which relies on the document tree. This nested hierarchy of elements provides for application of CSS properties from ancestor elements to decendant elements. These are mainly applicable to text-related and not box-related properties. Inhertiance provides for cleaner and more efficient code, as repetetive declarations can be avoided. This feature allows for faster loading web pages, with additional benefits to include decreased bandwidth and develpment costs.

Box Sizing

HTML elements are boxes that have four components:
  • margin
  • border
  • padding
  • content
